## 背景・目的

受講生は資格学習中の疑問を 1on1 のチャット(既存のチャット機能)でコーチに個別質問しているが、同じ疑問が複数受講生で繰り返されるため集合知が蓄積されない。コーチも類似質問への対応が重複し工数が膨らむ。

本チケットでは公開型の Q&A 掲示板を新規実装し、過去の質問・回答が他受講生にも参照可能な状態にする。受講生は自己解決率が向上し、コーチは未対応スレッドだけをフォローすればよい状態を達成する。集合知の蓄積によって学習効率向上とコーチ対応工数削減を同時に実現する中核機能。

## ユーザーストーリー

- **受講生(student)として**、公開済資格すべての掲示板に質問を投稿し、コーチや他受講生から回答を得たい。なぜなら、自分の疑問を集合知として解決したいから。
- **受講生として**、他受講生の質問・回答を閲覧したい。なぜなら、過去のスレッドから自己解決のヒントを得たいから。
- **受講生(スレッド投稿者)として**、自分の質問を解決済にマークしたい。なぜなら、未解決の質問と解決状態を区別したいから。
- **コーチ(coach)として**、担当資格の未対応スレッドを一覧から消化し、回答したい。なぜなら、複数受講生を効率的にフォローしたいから。
- **管理者(admin)として**、全資格の不適切な投稿をモデレーション削除したい。なぜなら、コミュニティの健全性を保ちたいから。

## 要件

### スレッドの管理(投稿は受講生のみ)

- 受講生による質問スレッドの新規投稿(資格を選んでタイトルと本文を入力)
- 投稿者本人によるスレッド編集(資格は変更できない)
- 投稿者本人によるスレッド削除(回答が付いている場合は削除できない)
- 投稿者本人による解決済 / 未解決の切り替え

### 回答の管理

- 受講生・コーチによる回答の投稿(管理者は回答できない)
- 投稿者本人による回答の編集 / 削除

### 一覧表示 / 検索 / 詳細閲覧

- スレッド一覧の表示(新着順、ページネーションあり)
- 資格・解決状態・キーワードによる絞り込み(キーワードはタイトル・本文・回答本文を対象に検索)
- スレッド詳細で配下の回答を一覧表示
- 一覧表示はスレッド件数が増えても取得時間が線形に増えないように関連データを効率的に取得する

### 管理者モデレーション

- 管理者は専用画面から、公開停止中の資格を含む全資格のスレッドを横断的に閲覧できる
- 管理者は任意のスレッド・回答を削除できる(内容の編集や解決マークの代行はできない)

### アクセス制御(機能群共通)

- 受講生は公開済資格すべてのスレッドを閲覧・投稿できる
- コーチは担当資格のスレッドのみ閲覧・回答でき、担当外の資格は操作できない
- 公開停止中の資格のスレッドは受講生・コーチには見えない(管理者は閲覧できる)
- 受講中の受講生・コーチのみアクセスできる

### 入力検証

- スレッドの資格: 必須 / 公開済の資格のみ選択可
- スレッドのタイトル・本文、回答の本文: 必須 / 文字数制限あり / 空白のみの入力は不可

## スコープ外

- 添付ファイル(画像 / PDF)対応 — テキストのみ
- Section / Question 等の教材階層への紐付け — 資格紐付けのみ
- ベスト回答指定 / ネスト回答 / 編集履歴 / いいね・投票 / タグ / メンション通知
- 全文検索エンジン — 部分一致検索のみ
- 管理者によるスレッド・回答の内容編集 / 解決マーク代行
- 同時編集競合の楽観ロック
- 論理削除 / 削除済表示トグル — 物理削除のみ(削除履歴は保持しない)
- 1on1 チャット機能(既存)
- **回答受信時の通知発火** — 本チケットでは通知関連を一切扱わない
- **未対応件数のサイドバーバッジ表示** — 本機能は提供しない

## インターフェース

**公開エンドポイント**(受講生 / コーチ):

| HTTP | パス | 認可 |
|---|---|---|
| GET | `/qa-board` | 受講生 / コーチ |
| GET | `/qa-board/create` | 受講生のみ |
| POST | `/qa-board` | 受講生のみ |
| GET | `/qa-board/{thread}` | 受講生 / コーチ |
| GET | `/qa-board/{thread}/edit` | 投稿者本人のみ |
| PATCH | `/qa-board/{thread}` | 投稿者本人のみ |
| DELETE | `/qa-board/{thread}` | 投稿者本人のみ |
| POST | `/qa-board/{thread}/resolve` | 投稿者本人のみ |
| POST | `/qa-board/{thread}/unresolve` | 投稿者本人のみ |
| POST | `/qa-board/{thread}/replies` | 受講生 / コーチ |
| GET | `/qa-board/{thread}/replies/{reply}/edit` | 投稿者本人のみ |
| PATCH | `/qa-board/{thread}/replies/{reply}` | 投稿者本人のみ |
| DELETE | `/qa-board/{thread}/replies/{reply}` | 投稿者本人のみ |

**管理者モデレーション**:

| HTTP | パス | 認可 |
|---|---|---|
| GET | `/admin/qa-board` | 管理者のみ |
| GET | `/admin/qa-board/{thread}` | 管理者のみ |
| DELETE | `/admin/qa-board/{thread}` | 管理者のみ |
| DELETE | `/admin/qa-board/{thread}/replies/{reply}` | 管理者のみ |

入口はサイドバーの「質問掲示板」項目。

## 初期データ

- 公開済の資格ごとに、未解決・解決済を混在させたスレッドを散布する
- 回答数（0 件 / 数件）と作成日時をばらつかせる（絞り込み・並び順・ページネーション・削除可否の動作を確認）
- 固定の受講生を投稿者にしたスレッドを用意する（「自分の質問」一覧や解決マークの動線を確認）
