# S-B-03 プランのマスタ管理

## 背景・目的

プランのデータ構造 + 初期データ + 管理画面 + 招待時 / プラン延長時にプランを参照する受講生側ロジックは既に含まれているが、管理画面側の認可・ルーティング・バリデーションのロジックが欠落しており画面が動作しない(404 になる)。運営側はプランの新設・受講期間 / 面談回数の改定・終売を直接操作できず、受講生招待時のプラン指定の前提も立ち上がらない。

本チケットでは、管理者(admin)がプランを CRUD でき、下書き → 公開中 → アーカイブ の状態遷移で招待時に選べるプランを運営側でコントロールできる管理画面を完成させる。削除には制限を設け、招待・受講中ユーザーとの参照整合性を守る。プラン受講モデルの前提となるマスタ管理画面であり、本チケットが揃わないと招待発行 / プラン延長 / 受講生ダッシュボードのプラン情報パネル等の後続フローの検証ができない。

## ユーザーストーリー

- **管理者(admin)として**、受講プランを CRUD したい。なぜなら、運営側で受講期間 / 初期付与面談回数 / 公開状態を自由にコントロールしたいから。
- **管理者として**、プランを下書きから公開へ、または招待画面の選択肢から外すためにアーカイブへ遷移させたい。なぜなら、プランのライフサイクル(新設 → 提供 → 終売)を自前でハンドリングしたいから。
- **管理者として**、運用中の参照を壊すプランの削除が起きない仕組みを期待する。なぜなら、受講中ユーザーの参照やプラン履歴が孤立して監査ができなくなるから。
- **受講生(student) / コーチ(coach)として**、管理者専用のプラン管理画面に直接アクセスしても拒否されることを期待する。なぜなら、運営側のマスタ管理画面に他ロールが入れる構成は事故の元だから。

## 要件

### プラン(管理者のみ)

- 一覧: プラン名のキーワード検索 + 状態フィルタ + ページネーション、各行に契約中の受講者数を表示
- 詳細: 基本情報 + 紐づく受講者一覧 + メタ情報(作成者 / 最終更新者 / 作成日時)を表示
- 新規作成: 初期状態は下書きで作成
- 編集: 基本情報(プラン名 / 説明 / 受講期間 / 初期付与面談回数 / 並び順)を更新。状態は本フォームでは変更しない
- 削除: 削除できる条件には制限がある(参照整合性を守る)

### 状態遷移(管理者のみ)

- 下書き・公開中・アーカイブの間でライフサイクルに沿って状態を切り替えられる
- アーカイブ後は招待画面 / プラン延長画面の選択肢から外れるが、受講中ユーザーの参照は維持される
- 不正な順序での状態変更はできない

### 入力検証(新規作成 / 編集)

- プラン名: 必須 / 文字数制限あり
- 説明: 任意 / 文字数制限あり
- 受講期間(日): 必須 / 適切な範囲の整数
- 初期付与面談回数: 必須 / 適切な範囲の整数
- 並び順: 任意 / 適切な範囲の整数

### アクセス制御(機能群共通)

- プラン管理画面の全操作は管理者のみ。受講生 / コーチはアクセス不可

## スコープ外

- **プラン価格情報の保持** — 決済は LMS 外で完結する方針のため、価格情報自体を持たない(確定済)
- **受講生招待時のプラン指定 / プラン情報の初期化**(受講プラン割当・受講期間・初期面談回数の確定)— 招待・オンボーディングフローの責務(本チケットでは扱わない)
- **プラン延長**(受講期間 + 面談回数の加算)— 既存機能。管理者のユーザー詳細画面の延長動線から呼び出す(本チケットでは扱わない)
- **期限満了による自動卒業** — 既存機能(本チケットでは扱わない)
- **プラン履歴の表示**(プラン割当 / 延長 / 期限満了の履歴) — 既存の履歴記録ロジックを利用するのは別画面の責務(本チケットの詳細画面では受講者一覧のみ表示)
- **期限切れ判定 / 残日数算出** — 既存の判定ロジックを別の機能(受講生ダッシュボード等)が利用する
- **受講生 / コーチ向けのプラン情報表示** — ダッシュボード / プロフィール側の責務
- **並び順をドラッグ&ドロップで操作する UI** — 並び順は数値入力で十分

## インターフェース

| HTTP | パス | 認可 |
|---|---|---|
| GET | `/admin/plans` | 管理者のみ |
| GET | `/admin/plans/create` | 管理者のみ |
| POST | `/admin/plans` | 管理者のみ |
| GET | `/admin/plans/{plan}` | 管理者のみ |
| GET | `/admin/plans/{plan}/edit` | 管理者のみ |
| PUT | `/admin/plans/{plan}` | 管理者のみ |
| DELETE | `/admin/plans/{plan}` | 管理者のみ |
| POST | `/admin/plans/{plan}/publish` | 管理者のみ |
| POST | `/admin/plans/{plan}/archive` | 管理者のみ |
| POST | `/admin/plans/{plan}/unarchive` | 管理者のみ |

受講生 / コーチが直接アクセスしても拒否される。

**画面**: 管理画面(`resources/views/plan/management/` 配下)は提供済み(本チケットでの画面変更は不要)。

## 初期データ

- プランの初期データは既に含まれている
- 下書き / 公開中 / アーカイブの各状態のプランが揃っている（状態フィルタ・状態遷移ボタンの活性条件を確認）
- 受講者が紐づくプランと紐づかないプランの両方がある（削除ガードの動作を確認）
