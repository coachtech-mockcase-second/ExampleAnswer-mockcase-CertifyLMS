## 背景・目的

コーチは面談予約・面談に紐づく議事メモは記録できるが、面談以外の日常観察(「最近 chat の応答が遅れている」「Q&A で躓いた論点」「次回面談で確認したい事項」等)を構造化して残す場所がない。複数受講生を持つコーチは私的なメモ帳や別ツールに分散管理することになり、引き継ぎや管理者監督が困難になっている。

本チケットでは、コーチが担当資格に登録した受講生の受講登録単位でメモを管理でき、時系列に履歴を残せる状態にする。複数コーチ体制下では各コーチが自分のメモのみ編集 / 削除でき、管理者は全コーチのメモを越境管理できる。受講生本人にはメモが完全に見えない(コーチ → 管理者の業務記録の性質を保つ)。担当コーチが受講生の状況を継続観察するための業務記録基盤であり、本機能が整うとコーチ間の知見共有が促され、担当変更時の引き継ぎも円滑になる。

## ユーザーストーリー

- **コーチ(coach)として**、担当資格に登録した受講生の受講登録単位でメモを追加・編集・削除したい。なぜなら、面談以外の日常観察を構造化して残し、フォローの質を高めたいから。
- **コーチとして**、他コーチが書いたメモも閲覧したい。なぜなら、複数コーチ体制での申し送り事項を相互に把握したいから。介入(編集 / 削除)はせず、自分のメモのみ自分で管理する。
- **管理者(admin)として**、任意の受講生に対する全コーチのメモを閲覧・編集・削除したい。なぜなら、運用上の不適切記述の是正やコーチ離任時のメモ管理を一元的に扱いたいから。
- **管理者として**、自分自身もコーチと同じフォーマットでメモを残せることを期待する。なぜなら、運営観察も同じ場所に記録した方が一貫しているから。
- **受講生(student)として**、自分の受講登録配下にコーチ / 管理者が残したメモが自分には一切表示されないことを期待する。なぜなら、メモは業務記録であり、受講生に見えると素直な観察ができなくなるから。

## 要件

### メモの管理(コーチ + 管理者のみ操作可、受講生は閲覧含めすべて拒否)

- 追加: コーチは担当資格に登録した受講生の受講登録に本文を入力して追加でき、追加後は元の画面に戻る。作成者として操作者が記録される
- 追加(管理者): 管理者は任意の受講登録にメモを追加でき、作成者には管理者が記録される
- 編集: コーチは自分が作成したメモのみ本文を更新できる。管理者は任意のメモを更新できる
- 削除: コーチは自分が作成したメモのみ削除できる。管理者は任意のメモを削除できる。削除すると履歴は残らない
- 作成者の不変性: 編集しても作成者は変わらない

### メモの閲覧

- コーチ: 担当資格に登録した受講生の受講登録詳細画面でメモ一覧を時系列で閲覧でき、自分のメモには編集 / 削除の操作が出る(他コーチのメモは閲覧のみ)
- 管理者: 任意の受講登録詳細画面でメモ一覧を閲覧でき、全メモを編集 / 削除できる
- 受講生本人: 自分の受講登録詳細画面にメモのセクション自体が現れない

### 入力検証

- 本文には文字数制限があり、空では追加できない

### アクセス制御(機能群共通)

- メモの閲覧 / 追加 / 編集 / 削除はコーチ(担当資格)と管理者のみ。受講生は閲覧含めすべて拒否
- 担当していない資格の受講登録に対するコーチのメモ操作は拒否
- 親の受講登録が削除された場合、その配下のメモは一覧から除外される

## スコープ外

- **受講生本人がメモを閲覧する動線** — 業務記録の性質を保つため、受講生は閲覧含め全拒否
- **メモの検索 UI** — 時系列閲覧のみ
- **コーチ間のノート個別非公開フラグ** — 他コーチのメモは常に閲覧可、個別非公開モードは持たない
- **メモへのコメント / リアクション機能** — 単純な本文のみ、対話機能なし
- **メモの達成マーク / ステータス管理** — メモはフラットな本文のみ
- **メモの復元 UI** — 削除のみ提供
- **添付ファイル(画像 / PDF)** — 本文テキストのみ
- **コーチ → 受講生宛のメッセージ** — 既存のチャット機能の責務、本機能は内部記録専用
- **コーチ離任時のメモ自動委譲 / 移管** — 自動的な作成者書き換えは行わない

## インターフェース

| HTTP | パス | 認可 |
|---|---|---|
| POST | `/enrollments/{enrollment}/notes` | 担当コーチ / 管理者 |
| GET | `/enrollment-notes/{note}/edit` | 作成者本人 / 管理者 |
| PATCH | `/enrollment-notes/{note}` | 作成者本人 / 管理者 |
| DELETE | `/enrollment-notes/{note}` | 作成者本人 / 管理者 |

全ルート認証必須。メモ一覧と追加フォームは受講登録詳細画面に埋め込み、編集は専用ページへ遷移する。

## 初期データ

- 複数の受講生・複数の資格・複数のコーチにまたがってメモを投入する
- 自分が作成したメモと他コーチが作成したメモを混在させる（編集 / 削除の出し分け・管理者の越境操作を確認）
- コーチの担当資格と担当外資格の両方の受講登録があり、受講生本人の受講登録にもメモがある（担当外での閲覧拒否・受講生にはメモが見えない分離を確認）
