# S-B-08 お知らせ配信（管理者）

## 背景・目的

既存の通知基盤は業務イベント(chat 受信 / Q&A 回答 / 面談予約 / 面談キャンセル)に紐づく **受動的な通知発火** のみで、運営側が能動的に受講生集合へメッセージを届ける手段がない。メンテナンス予告 / 重要更新 / 学習キャンペーン告知などの運営連絡を chat 個別送信や外部メールツールで代替する状況。

本チケットは管理者(admin)が能動的に受講生集合へ一斉配信できるお知らせ機能を新規実装する。配信対象は「全受講生」「資格指定」「ユーザー指定」の 3 種類から選択でき、既存の通知基盤を利用して各受講生にアプリ内通知とメールで届ける。配信は不可逆(再配信 / 編集 / 取消なし)で、配信履歴は管理者専用画面で時系列に閲覧できる。本機能が揃うと外部メールツールへの依存が減り、運営からの能動連絡を LMS 内に集約する **運営コミュニケーション基盤** が成立する。

## ユーザーストーリー

- **管理者(admin)として**、全受講中の受講生に対して一斉お知らせを配信したい。なぜなら、メンテナンス予告 / 重要更新 / 学習キャンペーン告知を運営から直接届けたいから。
- **管理者として**、指定資格に登録した受講生だけに絞ってお知らせ配信したい。なぜなら、資格固有の試験変更 / 教材更新を該当受講生だけに届けたいから。
- **管理者として**、指定 1 ユーザーだけにお知らせ配信したい。なぜなら、個別のフォロー連絡を通知として残したいから。
- **管理者として**、配信履歴を時系列で閲覧し、各お知らせの配信件数 / 配信時刻を確認したい。なぜなら、配信実績を後から監査できる状態にしたいから。
- **管理者として**、誤配信防止のため再配信 / 編集 / 取消ができない設計を期待する。なぜなら、メール配信は取り消せない不可逆性を持つため、UI 上も同じ不可逆性で運用責任を明確化したいから。
- **受講生(student)として**、運営からのお知らせを通常の通知一覧で業務通知と並列に受信したい。なぜなら、運営連絡を見落としたくないから。

## 要件

### 配信作成・実行(管理者のみ操作可)

- 配信作成フォームで、タイトル / 本文 / 配信対象 を入力して配信する
- 配信を実行すると、対象の受講生それぞれにアプリ内通知とメールが届き、配信実績が記録される

### 配信対象の 3 種類

- 配信対象は 全受講生 / 資格指定 / ユーザー指定 の 3 タイプから選ぶ

### 配信履歴閲覧(管理者のみ操作可)

- 配信履歴の一覧を時系列で閲覧できる
- 各お知らせの詳細を閲覧できる

### 配信の不可逆性

- 一度配信したお知らせは、再配信 / 編集 / 取消ができない

### 共通の振る舞い

- 利用状態やロールによって、配信対象に含まれないユーザーがいる
- 配信されたお知らせは既存の通知基盤に乗り、各受講生の通知一覧 / 未読バッジ / メール のすべてに届く
- 受講生が通知をクリックすると通知詳細ページに移り、お知らせ本文の全文を読める

## スコープ外

- **再配信 / 編集 / 取消** — 配信の不可逆性を仕様として確定
- **コーチ向けの配信** — 配信対象は受講生のみ
- **配信予約 / スケジュール送信** — MVP 外、配信は即時のみ
- **配信ターゲット内の追加絞り込み(複数資格 AND / 学習進捗フィルタ等)** — 単純な 3 種類のみ
- **配信前の対象件数プレビュー** — MVP 外
- **お知らせの優先度フラグ / 既読率追跡 / 開封確認** — MVP 外
- **添付ファイル / リッチテキスト本文** — プレーンテキストのみ
- **お知らせ専用の受講生向け独立画面** — 受講生は通知詳細ページで本文全文を読む
- **配信失敗のリトライ** — 同期送信のためエラー時は管理者が再操作

## インターフェース

| HTTP | パス | 認可 |
|---|---|---|
| GET | `/admin/announcements` | 管理者のみ |
| GET | `/admin/announcements/create` | 管理者のみ |
| POST | `/admin/announcements` | 管理者のみ |
| GET | `/admin/announcements/{announcement}` | 管理者のみ |
| GET | `/notifications/{notification}` | 認証ユーザー(自分宛の通知のみ) |

管理画面側のルートは管理者のみ。受講生の閲覧面となる通知詳細ページは、受信者本人のみが開ける。お知らせには編集 / 削除 / 再配信のルートを設けない。

**画面**: 管理画面(`resources/views/announcement/management/` 配下)と通知詳細ページ(`resources/views/notifications/show.blade.php`)は提供済み(本チケットでの画面変更は不要)。

## 初期データ

- 配信対象の 3 種類（全受講生 / 資格指定 / ユーザー指定）それぞれのお知らせを投入する
- 各お知らせに紐づく受講生の通知も投入する（配信履歴の一覧・詳細と受講生側の受信を確認）
