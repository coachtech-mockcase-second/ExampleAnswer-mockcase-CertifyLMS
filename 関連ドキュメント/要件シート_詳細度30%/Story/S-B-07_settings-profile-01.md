## 背景・目的

ユーザーは招待時に管理者から登録されるが、その後の自己情報(氏名表記の変更 / 自己紹介の追記 / アイコン画像のアップロード / パスワード変更)を更新する画面が存在しない。受講生は学習者としての自分の見え方をコントロールできず、コーチも担当する受講生に対する自己紹介を整えられず、管理者もパスワード変更などの基本セキュリティ動線がない。

本チケットでは、全ロールが共通の設定画面から自分の情報を更新できる状態にする。プロフィール編集 / パスワード変更 / アバター画像変更 をタブ切替で完結させ、コーチには固定面談 URL の編集を追加する。全ロール共通の基本機能として本 LMS の最低限のユーザー体験を整える土台となる。本チケットが揃わないと、運営側は些細な情報更新依頼まで管理者操作で代行することになり運用コストが増大する。

## ユーザーストーリー

- **受講生(student)として**、自分の氏名 / 自己紹介 / アイコン画像を変更したい。なぜなら、学習プロフィールを自分の状況に合わせて整えたいから。
- **受講生として**、パスワードを安全に変更したい。なぜなら、定期的な変更や漏洩懸念時の対応を自己完結したいから。
- **コーチ(coach)として**、上記に加え固定面談 URL を編集したい。なぜなら、面談ツール(Google Meet / Zoom 等)の URL を変更したときに自己更新したいから。
- **管理者(admin)として**、プロフィール表示 / パスワード変更 / アバター画像変更 を全ロール共通の動線で利用したい。なぜなら、管理者も同じ自己管理ニーズを持つから。
- **修了済(graduated)の受講生として**、プラン機能はロックされても自分のプロフィール / パスワード変更 / アバター更新は引き続き使いたい。なぜなら、修了後にもアカウント情報を保守する必要があるから。

## 要件

### プロフィール表示・編集(全ロール)

- 本人の氏名 / メール / 自己紹介 / アイコン画像 / ロール / アカウント状態を確認できる
- 氏名と自己紹介を編集できる。メールはこの画面からは変更できない
- 修了済受講生もこの画面を利用できる

### コーチ固定面談 URL(コーチのみ)

- コーチは自分の編集フォームから固定面談 URL を更新できる
- 受講生 / 管理者には固定面談 URL の入力欄が現れず、操作もできない

### パスワード変更(全ロール)

- 現在のパスワードを確認したうえで、新しいパスワードに変更できる

### アバター画像(全ロール)

- アイコン画像をアップロードでき、扱える画像の形式・サイズには制限がある
- アイコン画像を削除すると、アイコン未設定の表示に戻る

### タブ切替・共通

- プロフィールとパスワードをタブで切り替えて操作でき、リロード後も同じタブで再表示される
- 入力に不備があるときは、その場で内容を直せるよう案内が表示される
- 自分以外のユーザーの情報は更新できない

## スコープ外

- **面談設定タブ**(コーチ専用、面談可能時間枠カレンダー)— 本チケットでは扱わない
- **Google Calendar 連携の連携 / 解除 UI**(コーチ専用) — 本チケットでは扱わない
- **メールアドレスの変更動線** — 管理者経由のみ
- **自己退会動線** — 管理者経由のオペレーションに集約、LMS 内に動線なし
- **自己ロール変更** — 管理者経由のみ
- **2FA / IP 制限 / ログイン履歴の詳細管理** — MVP 外
- **API トークンの発行 / 失効 UI** — 不採用
- **学習時間目標の編集 UI** — 別画面の責務
- **通知設定 UI(種別 × チャネル ON/OFF)** — 不採用
- **プラン情報表示 / 追加面談購入 CTA** — ダッシュボードに集約
- **アバター画像の自動リサイズ / 圧縮** — 採用しない

## インターフェース

| HTTP | パス | 認可 |
|---|---|---|
| GET | `/settings/profile` | 全ロール(本人) |
| PATCH | `/settings/profile` | 全ロール(本人のみ) |
| POST | `/settings/avatar` | 全ロール(本人のみ) |
| DELETE | `/settings/avatar` | 全ロール(本人のみ) |
| PUT | `/settings/password` | 全ロール(本人のみ) |

全ルート認証必須。ロールによる制約はなく、修了済受講生も利用できる。

## 初期データ

- 各ロール（受講生 / コーチ / 管理者）と修了済の受講生が揃っている（各ロールでの表示・編集を確認）
- アバター設定済 / 未設定のユーザーが混在している（アイコンの差し替え・削除・イニシャル表示を確認）
