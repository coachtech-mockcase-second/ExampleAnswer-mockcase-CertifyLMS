# B-B-13 オンボーディングで確認用パスワードが一致しなくても登録できる

## メタ情報

| 項目 | 値 |
|---|---|

| Feature 連番 | `auth-03` |
| Feature | auth(オンボーディング) |
| 種別 | Bug |
| サブカテゴリ | データ(バリデーション) |
| 難易度 | Basic |
| 工数 (h) | 2 |
| 依存チケット | (なし) |

## 概要

招待を受けたユーザーがオンボーディングでパスワードを設定する際、パスワードと確認用パスワードが一致していなくても登録が通ってしまう。確認用入力の突き合わせが行われないため、入力ミス(typo)に気づけないまま意図しないパスワードが設定される。

## 再現手順

**前提**: 有効な招待 URL を受け取った招待中ユーザー。

1. オンボーディング画面で、名前と「パスワード」「確認用パスワード」を入力する
2. 「パスワード」と「確認用パスワード」にわざと異なる値を入力して送信する
3. → 入力エラーにならず、オンボーディングが完了してしまう

## 期待する結果

「パスワード」と「確認用パスワード」が一致しない場合は入力エラー(422)になり、オンボーディングが完了しない。両者が一致したときのみ登録が進む。

## 実際の結果

確認用パスワードとの一致チェックが行われず、両者が異なっていても登録が成立する。入力ミスを検出できないまま、利用者の意図しないパスワードが設定される。
