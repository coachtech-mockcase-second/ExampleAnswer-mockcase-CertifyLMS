# B-B-12 オンボーディング完了後に再ログインできなくなる

## メタ情報

| 項目 | 値 |
|---|---|
| チケット ID | `B-B-12` |
| Feature 連番 | `auth-02` |
| Feature | auth(招待 → オンボーディング) |
| 種別 | Bug |
| サブカテゴリ | 機能(状態遷移) |
| 難易度 | Basic |
| 工数 (h) | 3 |
| 依存チケット | (なし) |

## 概要

招待を受けたユーザーがオンボーディング(初回の名前・パスワード設定)を完了しても、利用状態が「招待中」のまま在籍中に切り替わらない。完了直後は自動ログインで画面に入れるが、一度ログアウトすると以降ログインできず、プラン機能にもアクセスできなくなる。

## 再現手順

**前提**: 有効な招待 URL を受け取った招待中ユーザー。

1. 招待 URL からオンボーディング(名前・パスワード設定等)を完了する
2. 完了直後は自動ログインでダッシュボードに入れる
3. 一度ログアウトし、設定したメールアドレス・パスワードで改めてログインを試みる
4. → ログインできない(認証が通らない)。教材などのプラン機能にもアクセスできない

## 期待する結果

オンボーディングを完了すると利用状態が「招待中」から「受講中(在籍中)」に切り替わり、以降は設定したメールアドレス・パスワードで再ログインでき、プラン機能も利用できる。

## 実際の結果

オンボーディング完了後も利用状態が「招待中」のまま残るため、再ログインが拒否され、在籍ユーザー向けの機能(プラン機能)もブロックされる。完了直後の自動ログインだけは通るため、問題が後から顕在化する。

## 受け入れ条件

- [ ] オンボーディング完了後に利用状態が在籍中(受講中)へ切り替わり、ログアウトしても設定した認証情報で再ログインできる
- [ ] オンボーディング完了後、在籍ユーザー向けのプラン機能にアクセスできる

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/UseCases/Auth/OnboardAction.php`(オンボーディング完了ユースケース)。利用状態 `UserStatus`(`Invited` → `InProgress`)の更新を担う。
- **仕込み内容**: 完了処理でユーザーの利用状態を在籍中(`UserStatus::InProgress`)へ更新する設定が抜けている。状態遷移の監査ログ(`UserStatusChangeService::record`)は遷移前 status を参照するため先に呼ばれログだけは残るが、実際の status カラムは `Invited` のまま残り、ログと状態が食い違う。ログイン認証(Fortify `AuthenticateUserUsing`、在籍中 `in_progress` / 修了 `graduated` のみ通過)と `EnsureActiveLearning`(在籍中のみ通過)が `Invited` を弾くため、完了直後の自動ログイン(`Auth::login`)だけは通り、再ログイン以降に詰む。
- **修正範囲**: 完了処理でユーザーの利用状態を在籍中(`InProgress`)へ更新する設定を戻す。`OnboardAction` 内 1 行(更新属性への status 付与)で完結。

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 完了直後は入れるのに再ログインできないのはなぜか? | 完了処理が完了直後にそのまま自動ログインさせるため。利用状態が「招待中」のままだと、ログアウト後の通常ログインや在籍機能ガードで弾かれる |
| 遷移後の利用状態は? | 「受講中(在籍中)」。ログイン認証と在籍ユーザー向け機能ガードがこの状態を要求する |
| ログインが拒否されるときのユーザー体験は? | ログイン画面に戻り、認証情報が正しくない旨の共通エラーが表示される(状態の詳細は伏せる) |
| 状態遷移とパスワード設定はどちらが先か? | 同じ完了処理(トランザクション)の中で確定する。本 Bug は「状態遷移だけが抜けている」ケースで、パスワード設定自体は成立する |
