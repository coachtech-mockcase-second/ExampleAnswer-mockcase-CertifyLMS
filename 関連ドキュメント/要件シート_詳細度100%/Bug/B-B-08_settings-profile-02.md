# B-B-08 プロフィール更新後のリダイレクト先が間違っている

## 概要

受講生・コーチ・管理者が設定画面でプロフィールを更新すると、本来は設定画面(プロフィールタブ)に戻って更新結果を確認できるべきところを、ダッシュボードに遷移してしまう。更新自体は成功するが、編集を続けたいユーザーがその都度設定画面へ戻り直す必要があり、操作の流れが途切れる。

## 再現手順

**前提**: いずれかのロールでログイン済み。

1. 設定画面のプロフィールタブで、氏名や自己紹介などを変更して保存する
2. → プロフィールは更新される(成功メッセージは表示される)
3. しかし設定画面に戻らず、ダッシュボードに遷移してしまう

## 期待する結果

プロフィール更新に成功すると、設定画面のプロフィールタブにリダイレクトされ、更新後の内容と成功メッセージが表示される。

## 実際の結果

プロフィール更新に成功した後、設定画面ではなくダッシュボードにリダイレクトされ、更新結果を同じ画面で確認できない。

## 受け入れ条件

- [ ] プロフィール更新に成功すると、設定画面のプロフィールタブにリダイレクトされる
- [ ] リダイレクト後に更新成功のメッセージ(フラッシュ)が表示される

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/Http/Controllers/Settings/ProfileController.php` の `update()`(`PATCH /settings/profile`)
- **仕込み内容**: 更新成功時のリダイレクト先が、設定画面のプロフィールタブ(`settings.profile.edit` の `?tab=profile`)からダッシュボードに差し替えられている。成功フラッシュ自体は残るため「更新は成功するが画面が飛ぶ」状態になる。設定画面は複数タブ(プロフィール / パスワード / 等)を持ち、各保存は対応するタブ付き設定画面へ戻る設計のため、プロフィール更新の正しい戻り先はプロフィールタブ
- **修正範囲**: `update()` のリダイレクト先を設定画面のプロフィールタブに戻す。Controller のリダイレクト 1 箇所の修正で完結

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 更新後のリダイレクト先は具体的にどこか? | 設定画面のプロフィールタブ(`/settings/profile?tab=profile` 相当)。更新内容を同じ画面で確認できる状態に戻す |
| 更新成功メッセージは表示されるべきか? | 表示される。リダイレクト先(設定画面)で成功フラッシュが出る |
| パスワード変更・アバター更新など他の保存も同じ問題か? | 本 Bug の対象はプロフィール更新。他の保存操作は各タブ付き設定画面へ戻る挙動が正しい(修正時に他の保存のリダイレクト先も揃っているか確認すると良い) |
| どのロールで再現するか? | 設定画面は全ロール共通のため、受講生 / コーチ / 管理者いずれでも再現する |
