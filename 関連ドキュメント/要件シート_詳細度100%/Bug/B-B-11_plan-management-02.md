# B-B-11 プラン管理一覧の状態フィルタが正しく絞り込めない

## メタ情報

| 項目 | 値 |
|---|---|
| チケット ID | `B-B-11` |
| Feature 連番 | `plan-management-02` |
| Feature | plan-management |
| 種別 | Bug |
| サブカテゴリ | データ(クエリ条件誤り) |
| 難易度 | Basic |
| 工数 (h) | 3 |
| 依存チケット | `S-B-03`(プラン管理 Admin マスタ UI が前提) |

## 概要

管理者(admin)がプラン管理の一覧画面で状態フィルタを「公開中」に指定しても、公開中のプランが表示されず、代わりに下書きのプランが表示される。状態フィルタの絞り込み条件が指定値と食い違っており、フィルタが意図どおりに機能しない。

## 再現手順

**前提**: 管理者でログイン済み。公開中のプランと下書きのプランがそれぞれ 1 件以上存在する(初期データに含まれる)。

1. プラン管理の一覧画面を開く
2. 状態フィルタで「公開中」を選ぶ
3. → 公開中のプランが表示されず、下書きのプランが表示される(絞り込み結果が指定と食い違う)

## 期待する結果

状態フィルタで「公開中」を選ぶと公開中のプランのみ、「下書き」を選ぶと下書きのプランのみが表示される。フィルタの指定値と表示結果が一致する。

## 実際の結果

状態フィルタの指定に関わらず特定の状態(下書き)で絞り込まれてしまい、「公開中」を指定しても公開中プランが表示されない。

## 受け入れ条件

- [ ] 状態フィルタで「公開中 / 下書き / アーカイブ」を選ぶと、選んだ状態のプランのみが一覧に表示される
- [ ] 状態フィルタを指定しないときは全状態のプランが表示される(正常系が維持されている)
- [ ] 本チケットの機能に対するテスト (Unit / Feature 等) が実装されている

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/UseCases/Plan/IndexAction.php`(プラン一覧取得の status / keyword フィルタ適用)。※ Action 内のため Basic 範囲外。Controller 内完結で実装する受講生は `PlanController::index` の一覧取得処理が対象
- **仕込み内容**: 状態フィルタの絞り込み条件が、画面で指定された状態値ではなく特定の状態(下書き = `PlanStatus::Draft`)で固定されている(状態 Enum 値のコピペミス)。これにより「公開中」を指定しても下書きで絞り込まれる。「下書き」を選んだときだけ偶然一致するため気づきにくい
- **修正範囲**: 絞り込み条件を「フィルタで渡された状態値で `where('status', ...)` する」形に戻す。`PlanController::index` から `Plan\IndexAction` を辿り、絞り込みに固定値が書かれていることを発見する流れ。status 絞り込み 1 箇所の修正で完結

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 状態フィルタの選択肢は? | 公開中 / 下書き / アーカイブ(プランの 3 状態)。指定した状態のプランのみに絞り込む |
| フィルタなしのときの表示は? | 全状態のプランを表示(絞り込みなし)。並び順は公開中を優先する想定(本 Bug の対象は絞り込み条件であり、並び順は別論点) |
| 「下書き」を選んだときだけ正しく動くように見えるのはなぜか? | 絞り込み条件が下書きで固定されているため、「下書き」を選んだときだけ偶然一致する。他の状態を選ぶと食い違う |
| キーワード検索との併用はできるか? | できる。キーワード + 状態フィルタの併用が正しい挙動(本 Bug は状態フィルタの条件誤り) |
