## 概要

チャットの未読バッジ(件数表示)が、相手から届いた未読メッセージだけでなく、自分が送ったメッセージまでカウントしてしまう。その結果、相手からの新着が無くても未読バッジが表示され、件数が実態とずれる。

## 再現手順

**前提**: 受講生(student)でログイン済み。Seeder データで、自分が送ったメッセージが自分の最終既読時刻より後に残っているチャットルーム(相手からの新着メッセージは無し)が 1 つある状態(例: 自分が最後に発言したまま、そのルームをまだ開き直していない)。

1. ログイン後、サイドバーの chat バッジ、またはチャット画面のルーム一覧(rooms-pane)で当該ルームの未読件数を確認する
2. → 相手からの新着が無いにもかかわらず、自分が送ったメッセージの件数だけが未読としてカウントされ、バッジが表示される

## 期待する結果

未読バッジは「相手が送った、かつ自分がまだ読んでいない」メッセージのみをカウントする。自分が送ったメッセージは未読に含めない。相手からの新着が無いルームは未読 0 として扱われ、サイドバー / ルーム一覧のバッジに出ない。

## 実際の結果

未読集計に自分の発言も含まれ、相手からの新着が無くても、自分が送ったメッセージの件数だけ未読としてカウントされる。当該ルームが未読ルームとしてサイドバーに数えられ、ルーム別バッジにも件数が表示される。

## 受け入れ条件

- [ ] 未読バッジ(サイドバーの未読ルーム数 / チャット画面のルーム別未読件数)が、自分が送ったメッセージをカウントしない
- [ ] 相手から届いた未読メッセージは従来どおりカウントされ、ルームを開いて既読化すると未読から外れる(正常系が維持されている)
- [ ] 本チケットの機能に対するテスト (Unit / Feature 等) が実装されている

## 実装方針(参考)

> **粒度**: 業務語彙で原因要約 + 主要ファイルパス・クラス名・メソッド名を併記。修正コード片 / SQL 詳細は書かない(コードリーディングは受講生の作業)。

### 原因

- **主要ファイル**: `app/Services/ChatUnreadCountService.php`(個人別未読件数の集計)。集計結果はサイドバーバッジ(`SidebarBadgeComposer` → `roomCountForUser`)と、チャット画面のルーム別バッジ(`ChatRoomController::show` → `messageCountsByRoomForUser`)から利用される。**※ 集計が Service 内にあるため Basic 範囲外**。Basic 受講生が未読集計を Controller / View Composer 内に書いている場合は、その実装箇所(サイドバー = `SidebarBadgeComposer`、チャット画面 = `ChatRoomController::show`)を修正対象とする。
- **仕込み内容**: 未読集計の各メソッド(`messageCountInRoom` / `messageCountsByRoomForUser` / `roomCountForUser`)から「送信者が自分でない」絞り込み(`where('sender_user_id', '!=', $user->id)`)が抜けている。同じ条件が 3 メソッドに分散しているため、再現するには全メソッドから削除する必要がある。
- **修正範囲**: 各集計メソッドのクエリに「送信者が自分でない」条件を戻す(「自分の最終既読時刻より後」の未読時刻条件はそのまま維持)。3 メソッドに同種の条件があるため、漏れなく修正する。

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 未読としてカウントする条件は? | 「送信者が自分以外」かつ「自分の最終既読時刻(`ChatMember.last_read_at`)より後に投稿された」メッセージ。両方を満たすものだけが未読(最終既読時刻が未設定なら時刻条件は常に満たす) |
| 既読化はどのタイミングで起きるか? | チャットルームを開いたときに自分の最終既読時刻が現在時刻に更新される。加えてメッセージ送信時にも送信者自身の最終既読時刻が更新されるため、通常操作で「送信した直後に自分の発言で未読が増える」ことは起きにくい。本 Bug が顕在化するのは、最終既読時刻より後に自分の発言が残っているルーム(自分が最後に発言したまま開いていない等)の未読件数をサイドバー / ルーム一覧で見たとき |
| 未読バッジはどこに出るか? | 2 箇所。サイドバーの chat メニュー(未読のあるルーム数)と、チャット画面のルーム一覧(ルーム別の未読メッセージ件数、99 件超で「99+」表示)。どちらも同じ集計ロジック由来なので、両方で症状が出る |
| 自分が送ったメッセージは相手側では未読になるべきか? | なる。相手から見れば「相手(= 自分)が送った未読」として正しくカウントされる。本 Bug は「自分が自分の発言を未読として数えてしまう」点だけが誤り |
