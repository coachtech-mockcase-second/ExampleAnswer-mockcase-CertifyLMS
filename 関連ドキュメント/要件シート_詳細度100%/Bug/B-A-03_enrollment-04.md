# B-A-03 模試をキャンセルしても学習タームが基礎タームに戻らない

## 概要

受講生(student)が模試の受験セッションを作成 → キャンセル(未開始のまま破棄)した場合、本来は学習タームが **基礎ターム** に戻るべきだが、誤って **実践ターム** のままになる。一度模試を作成・キャンセルしただけで、進行中の模試が無くなっても永遠に「実践ターム」と判定される(教習所メタファーの「教習生 → 卒業検定挑戦中」が戻らない)。

## 再現手順

**前提**: 受講生でログイン済み、受講中の資格に紐づく公開模試が 1 つ以上ある。受講登録の現在のタームは「基礎ターム」。

1. 模試一覧から任意の模試を選び「受験セッションを作成する」を実行する → タームが「実践ターム」に切り替わる(これは正常)
2. 作成された受験セッション(未開始)の詳細画面で「キャンセル」を実行する → セッションがキャンセル状態に遷移
3. ダッシュボード or 受講登録詳細を開いて「現在のターム」を確認する
4. → 進行中・提出済・採点済の模試が 1 件も存在しないにもかかわらず、ターム表示が **実践ターム** のままになっている

## 期待する結果

受講生が抱える受験セッションの状態が **受験中 / 提出済 / 採点完了** のいずれでもなくなった時点で(= 進行中の模試が 1 件も存在しない状態に戻った時点で)、学習タームは **基礎ターム** に再判定される。未開始のまま破棄したセッション(キャンセル状態)は「進行中の模試」として扱わない。

## 実際の結果

タームの再判定対象に **キャンセル状態のセッションも含まれている** ため、過去に 1 回でも模試セッションを作成・キャンセルした受講生は、進行中の模試が 1 件も無くても「実践ターム」と判定され続ける。基礎ターム表示への巻き戻しが永続的に起きない。

## 受け入れ条件

- [ ] 受験セッションを「作成 → キャンセル」したのち進行中の模試が 1 件も無い状態では学習タームが **基礎ターム** に再判定され、受験中 / 提出済 / 採点完了 のセッションが 1 件でも残っている場合は **実践ターム** のままになる(キャンセル済セッションとの混在ケースも含む)
- [ ] 受験セッションを「作成 → 受験開始」した瞬間に学習タームが **実践ターム** に切り替わる(既存挙動が維持されている)
- [ ] 現在のタームと再判定結果が一致する場合、不要な更新が走らない(既存挙動が維持されている)
- [ ] 本チケットの機能に対するテスト (Unit / Feature 等) が実装されている

## 実装方針(参考)

> **粒度**: 業務語彙で原因要約 + 主要ファイルパス・クラス名・メソッド名を併記。修正コード片 / SQL 詳細は書かない(コードリーディングは受講生の作業)。

### 原因

- **主要ファイル**: `app/Services/TermJudgementService.php` の `recalculate(Enrollment)`(学習ターム判定の集約 Service)。この Service は、受験セッションの状態変化を伴う `MockExamSession\{Start,Submit,Destroy}Action` が状態変更と同一トランザクション内で呼び出す
- **仕込み内容**: ターム判定で「進行中の模試あり」とみなす状態集合に、本来含めるべきでない **キャンセル状態が誤って混入** している。これにより一度でも受験セッションを作成 → キャンセルした受講生は、進行中の模試が 0 件でも実践ターム判定が続き、基礎タームに戻らない。判定対象に含めるべきは 受験中 / 提出済 / 採点完了 の 3 状態のみで、未開始 / キャンセル は対象外(※ Service 内のため Basic 範囲外)
- **修正範囲**: `recalculate` 内の状態集合からキャンセル状態を除外するだけで完結する。「進行中の模試が 1 件でもあるか」の存在判定 / 現在のタームと一致する場合に更新をスキップする分岐 はそのまま維持する

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 「実践ターム」と判定する受験セッションの状態は何ですか? | 受験中 / 提出済 / 採点完了 の 3 状態。これらが 1 件でも存在すれば実践ターム |
| 「未開始」は実践タームに含めないのですか? | 含めない。実践ターム入りのトリガは「受験開始」の瞬間。セッション作成だけで実践タームに入れると、作って放置しただけの受講生も基礎ターム表示に戻れなくなる |
| 「キャンセル」を含めないのは何故ですか? | キャンセルは「実践タームに足を踏み入れたが破棄した」状態。判定対象に入れると一度キャンセルした受講生が永久に実践ターム扱いになる(= 本 Bug)。教習所メタファーで「卒業検定の挑戦履歴を破棄したら教習生(基礎ターム)に戻る」自然な振る舞い |
| 「採点完了」を含めるのは何故ですか? | 教習所メタファーで「卒業検定挑戦中 ⊇ 結果確認」とみなし、採点完了後も再受験準備の期間として実践タームに留める。受験履歴の有無を「いま実践フェーズか」の判定材料にする設計 |
| 一度実践タームに入った後、基礎タームに戻ることはありますか? | ある。進行中(受験中 / 提出済 / 採点完了)の模試が 0 件になった瞬間に基礎ターム判定が走る |
| ターム判定はどのタイミングで走りますか? | 受験セッションの状態変化を伴う操作(受験開始 / 提出 / キャンセル)から、状態変更と同一トランザクション内で自動的に呼び出される。受講生が手動でリフレッシュを要求する画面は存在しない |
| 複数の資格を受講している場合、片方の資格でのキャンセルが他資格のタームに影響しますか? | しない。タームは受講登録(受講生 × 資格)ごとに独立して保持される |
| 同じタームが連続して算出された場合、毎回更新が走りますか? | 走らない。現在のタームと新しい判定結果が一致したら更新をスキップする分岐があり、本 Bug の修正範囲ではこの分岐には触れないこと |
