# B-B-06 招待トークンが使い回せてしまう

## メタ情報

| 工数 (h) | 3 |

## 概要

招待メールに含まれる招待 URL(トークン)が、一度オンボーディングを完了した後も使い回せてしまう。本来は一度使った招待 URL は使用済みになって再利用できないはずだが、同じ URL から何度でもオンボーディングをやり直せる(名前・パスワードの再設定が繰り返せる)状態になっている。

## 再現手順

**前提**: 有効な招待 URL を 1 通受け取っている(招待中ユーザー)。

1. 招待 URL を開き、オンボーディング(名前・パスワード設定等)を完了する
2. 同じ招待 URL をもう一度開く
3. → 使用済みエラーにならず、オンボーディング画面が再び開けてしまい、再度登録内容を上書きできる

## 期待する結果

一度オンボーディングを完了した招待 URL に再アクセスすると、使用済みとして拒否され(410)、オンボーディングを再実行できない。

## 実際の結果

オンボーディング完了後も招待が「未使用」の状態のまま残るため、同じ URL から何度でもオンボーディングが可能で、登録内容を繰り返し上書きできてしまう。

## 受け入れ条件

- [ ] オンボーディングを完了した招待 URL に再アクセスすると、招待リンクが無効である案内画面が表示され、オンボーディングを再実行できない
- [ ] 未使用の有効な招待 URL からのオンボーディングは従来どおり完了できる(正常系が維持されている)
- [ ] 同梱 `tests/Feature/UseCases/Auth/OnboardActionTest.php::test_marks_invitation_accepted`(修正前は赤)が pass する

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/UseCases/Auth/OnboardAction.php`(オンボーディング完了ユースケース、`__invoke`)。冒頭の検証ガードは招待が未使用(`InvitationStatus::Pending`)かつ期限内であることを確認し、不正なら `App\Exceptions\Auth\InvalidInvitationTokenException`(410)を throw する。
- **仕込み内容**: 完了処理内の「招待を使用済み(`InvitationStatus::Accepted` + 受諾日時 `accepted_at`)に更新する」処理が抜けている。使用済み化が漏れると招待は `Pending` のまま残り、同じトークンで何度でも冒頭ガードを通過して再オンボードできる(名前・パスワードの再上書きが成立する)。
- **修正範囲**: オンボーディング完了時に招待を使用済み(`Accepted`)へ更新する処理を戻す。`OnboardAction` 内の数行(招待レコードの状態更新)で完結。

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 使用済み招待 URL に再アクセスしたときの HTTP ステータスは? | 410(Gone)。一度きりのトークンが消費済みであることを示す |
| 招待 URL には有効期限もあるのか? | ある(署名付き URL + 招待レコードの有効期限)。本 Bug の対象は「期限内でも完了後に使い回せる」点(使用済み化漏れ)であり、期限切れ判定とは別 |
| オンボーディング完了後、ユーザーはどうなるか? | 自動的にログインしてダッシュボードへ遷移。完了した招待は使用済みになり再利用不可 |
| 同じメールアドレスに再招待したい場合は? | 管理者が改めて招待を発行する(新しい招待 URL が発行される)。既存の使用済み URL は復活しない |
