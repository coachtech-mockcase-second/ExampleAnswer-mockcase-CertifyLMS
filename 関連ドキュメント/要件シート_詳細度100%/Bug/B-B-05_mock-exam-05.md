## 概要

管理者(admin)が模試マスタを作成・編集するとき、合格点に 100% を超える値(例: 150%)を設定できてしまう。合格点は本来 1〜100% の範囲のはずだが、上限の検証が抜けており 100% 超が通ってしまう。得点率は最大でも 100% なので、合格点が 100% を超える模試は満点(全問正解)でも合格基準に到達できず、全受講生(student)が不合格になる。

## 再現手順

**前提**: 管理者でログイン済み。受講生が受験できる公開資格が 1 つ以上ある。

1. 模試マスタの作成画面で、合格点に `150`(100 を超える値)を入力して保存する
2. → バリデーションエラーにならず、合格点 150% の模試が保存される
3. 受講生でその模試を受験し、全問正解(得点率 100%)で提出する
4. → 採点結果が **不合格** と判定される(100% < 150% のため、満点でも合格できない)

## 期待する結果

合格点は 1〜100% の範囲でのみ設定でき、100% を超える値(範囲外)を入力するとバリデーションエラーで弾かれて保存されない。合格点が適切に設定された模試では、得点率が合格点以上で合格・未満で不合格と判定される。

## 実際の結果

合格点 100% 超の模試が作成・保存でき、その模試では得点率が最大の 100%(満点)でも合格基準に届かないため、全受講生が不合格になる。

## 受け入れ条件

- [ ] 模試マスタの作成・編集で合格点に 100 を超える値を入力すると日本語のバリデーションエラーが表示され、保存されない(設定できるのは 1〜100 の範囲のみ)
- [ ] 合格点が適切に設定された模試では、得点率が合格点以上で合格・未満で不合格と判定される(正常系が維持されている)
- [ ] 同梱 `tests/Unit/Http/Requests/MockExam/StoreRequestTest.php::test_validation_fails`(データセット「passing_score 101 で 422」、修正前は赤)が pass する

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/Http/Requests/MockExam/StoreRequest.php` / `app/Http/Requests/MockExam/UpdateRequest.php`(模試マスタの作成 / 更新の入力検証)
- **仕込み内容**: 合格点(`passing_score`)の許容範囲の上限検証が抜けており、100 を超える値を受け付けてしまう。得点率は最大 100% なので合格点 100% 超は「満点でも合格できない」= 全員不合格を意味し業務上無意味だが、検証を通過して保存される。下限(1 以上)側は正しく弾かれる
- **修正範囲**: `passing_score` の許容範囲に上限 100 を戻し、100 を超える値(範囲外)を弾く。作成 / 更新の両方の入力検証を揃える

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 合格点の有効範囲は? | 1〜100 の整数(百分率)。101 以上や 0 以下は範囲外として弾く |
| 合格点 100% 超を許すとどうなるか? | 得点率は最大 100% なので、合格点が 100% を超えると満点でも合格基準に届かず、全受講生が無条件に不合格になる。合格点として無意味なので上限は 100 |
| 下限(0 以下)は弾かれているか? | 弾かれている(下限側の検証は正しい)。本 Bug は上限(100 超を許容)のみ |
| 既に保存された不正な合格点(100% 超)のデータは? | 本チケットの範囲は新規作成 / 更新時の入力検証の修正。既存の不正データは初期データ再投入(`migrate:fresh --seed`)で復旧する範囲を想定 |
| 合格判定の比較はどちら(以上 / 超過)か? | 得点率 ≧ 合格点 で合格(以上判定)。本 Bug の修正範囲はこの判定ロジックには触れず、合格点の入力検証のみ直す |
