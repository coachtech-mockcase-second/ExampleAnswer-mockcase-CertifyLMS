# B-B-09 受講登録していない資格の教材詳細を直リンクで閲覧できてしまう

## 概要

受講生(student)が、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)を、URL を直接指定すること(直リンク)で閲覧できてしまう。本来、受講生は自分が受講登録している資格の教材しか閲覧できないはずだが、登録外の資格の教材が露出する(IDOR)。

## 再現手順

**前提**: 受講生 A でログイン済み。受講生 A は資格 X に受講登録済みだが、資格 Y には未登録。資格 Y には公開済みの教材が存在する。

1. 受講生 A で、未登録の資格 Y に属する教材詳細(Part / Chapter / Section)の URL を直接開く
2. → 弾かれず、資格 Y の教材詳細が表示されてしまう
3. 受講登録済みの資格 X の教材は従来どおり表示される(対比用)

## 期待する結果

受講生が、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)の URL に直接アクセスすると 403 が返る。受講登録している(受講中 / 合格)資格の教材のみ閲覧できる。

## 実際の結果

受講登録の有無を確認しないまま教材詳細が表示され、URL を知っていれば未登録資格の教材を直リンクで閲覧できてしまう。

## 受け入れ条件

- [ ] 受講生は、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)を閲覧できない
- [ ] 受講登録済み(受講中 / 合格)の資格の教材詳細は従来どおり閲覧できる
- [ ] 同梱 `tests/Feature/Http/Learning/BrowseControllerTest.php::test_show_part_forbidden_for_non_enrolled_student`(修正前は赤)が pass する

## 実装方針(参考)

### 原因

- **主要ファイル**: `app/Http/Controllers/BrowseController.php` の `showPart()` / `showChapter()` / `showSection()`(受講生向け教材閲覧の各画面、`/learning/parts/{part}` ほか)。
- **仕込み内容**: 各 show メソッド先頭の認可チェック呼び出し(`$this->authorize('learning.part.view' / 'learning.chapter.view' / 'learning.section.view', ...)`)が抜けている。認可ロジック自体(専用の Gate / Policy で「対象資格を受講中 / 合格で受講登録しているか」を判定)は正しく実装済みだが、Controller がそれを呼んでいないため受講登録の判定がスキップされ、URL を直接叩けば未登録資格の教材が閲覧できる。
- **修正範囲**: 各教材閲覧メソッドに認可チェック呼び出しを戻す(各メソッド 1 行)。Policy / Gate 側のロジックは正しいため、Controller の認可呼び出し漏れの発見と復元が要点。

## 補足

### 想定 Q&A

| 質問 | 回答 |

| 未登録資格の教材にアクセスしたときの HTTP ステータスは? | 403。受講登録していない資格の教材は閲覧権限なしとして弾く |
