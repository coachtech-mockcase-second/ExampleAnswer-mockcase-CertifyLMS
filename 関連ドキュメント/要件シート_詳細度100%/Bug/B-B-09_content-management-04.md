## 概要

受講生(student)が、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)を、URL を直接指定すること(直リンク)で閲覧できてしまう。本来、受講生は自分が受講登録している資格の教材しか閲覧できないはずだが、登録外の資格の教材が露出する(IDOR)。

## 再現手順

**前提**: 受講生 A でログイン済み。受講生 A は資格 X に受講登録済みだが、資格 Y には未登録。資格 Y には公開済みの教材が存在する。

1. 受講生 A で、未登録の資格 Y に属する教材詳細(Part / Chapter / Section)の URL を直接開く
2. → 403 で弾かれず、資格 Y の教材詳細が表示されてしまう
3. 受講登録済みの資格 X の教材は従来どおり表示される(対比用)

## 期待する動作

受講生が、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)の URL に直接アクセスすると 403 が返る。受講登録している(受講中 / 合格)資格の教材のみ閲覧できる。

## 実際の動作

受講登録の有無を確認しないまま教材詳細が表示され、URL を知っていれば未登録資格の教材を直リンクで閲覧できてしまう。

## 受け入れ条件

- [ ] 受講生が、自分が受講登録していない資格の教材詳細(Part / Chapter / Section)の URL に直接アクセスすると 403 が返る
- [ ] 受講登録済み(受講中 / 合格)の資格の教材詳細は従来どおり閲覧できる(正常系が維持されている)

## 実装方針

> **参考設計の一例**。受け入れ条件を満たせれば実装手段は問わない。受講生は提供 PJ コード + ヒアリングで自分の調査・修正方針を組み立てる。

### 主要 URL

| メソッド | パス | 振る舞い |
|---|---|---|
| GET | `/learning/parts/{part}` | 教材閲覧: Chapter 一覧 |
| GET | `/learning/chapters/{chapter}` | 教材閲覧: Section 一覧 |
| GET | `/learning/sections/{section}` | 教材閲覧: Section 詳細(本文) |

### 原因箇所メモ

- 原因の主要ファイル: `app/Http/Controllers/BrowseController.php` の `showPart()` / `showChapter()` / `showSection()`
- 関連: 受講生向け教材閲覧の認可は専用の Gate / Policy(`learning.part.view` / `learning.chapter.view` / `learning.section.view`)で「対象資格を受講中 / 合格で受講登録しているか」を判定する。認可ロジック(Policy)自体は正しく実装済み
- 仕込み内容(Step 4 引き算): 各 show メソッドの先頭にある認可チェック呼び出し(`$this->authorize('learning.*.view', ...)`)を削除する。認可呼び出しが無くなると、受講登録の判定がスキップされ、URL を直接叩けば誰でも教材を閲覧できる
- 受講生が辿るべき修正範囲: 各教材閲覧メソッドに認可チェック呼び出しを戻す。Controller の `$this->authorize()` 呼び出し(各メソッド 1 行)で完結。Policy 側のロジックは既に正しいため、Controller の認可呼び出し漏れを発見するのが要点

| 未登録資格の教材にアクセスしたときの HTTP ステータスは? | 403。受講登録していない資格の教材は閲覧権限なしとして弾く |
