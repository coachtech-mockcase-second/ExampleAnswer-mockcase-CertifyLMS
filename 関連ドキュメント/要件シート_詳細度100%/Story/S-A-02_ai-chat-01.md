## 背景・目的

本チケットでは、受講生が学習中いつでも AI に相談できる **AI 相談機能(AI チャットボット)** を新規に構築する。全画面共通のフローティングウィジェットと専用のフル画面チャットの両方から、生成 AI(Gemini)に学習相談ができるようにする。

現状、受講生は教材や問題演習で詰まったときコーチ面談やコーチへの個別チャットで質問しているが、軽い疑問でもコーチ対応待ちで学習が止まる / 教材文脈を毎回テキストで説明し直す必要がある / 深夜・早朝はコーチが応答できない、といった体験ギャップがある。

そこで AI を即時応答の「学習補助」の一次窓口として常駐させ、教材や対象資格の文脈は自動付与で迷子にならず、コーチ面談は本当に必要な深い相談に集中させられる状態にする。狙いは、学習継続率の向上(詰まり時の即時アンロック)とコーチ稼働の本質的相談への集約。

本機能は外部 LLM 連携(Gemini API)/ 日次レート制限 / 同期応答とその失敗時フォールバックといった、実務で頻出する外部 API 連携の要素を含む。

## ユーザーストーリー

- **受講生(student)として**、学習中はどの画面からでも右下のボタンから AI 相談を開き、その場で質問したい。なぜなら、コーチの応答を待たず、詰まったその瞬間に疑問を解消して学習を止めたくないから。
- **受講生として**、教材を読んでいる最中に AI 相談を開いたら「今読んでいる Section」を踏まえて回答してほしい。なぜなら、毎回どの教材のどこで詰まっているかを説明し直す手間を省きたいから。
- **受講生として**、教材を開いていないときでも、自分が今目指している資格を踏まえて回答してほしい。なぜなら、受講中の資格に沿った的確なアドバイスがほしいから。
- **受講生として**、過去の相談履歴を後から見返したい。なぜなら、AI の回答を学習ノート代わりに参照したいから。
- **受講生として**、AI が一時的に失敗しても自分の質問メッセージが履歴に残ってほしい。なぜなら、何を聞いたかが分かり、同じ内容を送り直して再質問しやすいから。
- **受講生として**、相談の会話タイトルを後から編集・削除したい。なぜなら、相談を整理してあとから探しやすくしたいから。
- **コーチ(coach) / 管理者(admin)として**、自分の画面に AI 相談 UI が表示されないでほしい。なぜなら、受講生専用機能であり、自分の業務には関係ないから。

## 要件

### 会話の管理(学習中受講生・会話オーナーのみ)

- 会話の新規作成 / 詳細表示 / 削除(配下メッセージも連動削除)
- 最新会話への自動遷移(会話が 0 件なら空状態画面を表示)
- 会話タイトルの扱い:
  - 新規作成時は暫定タイトル(「新規相談」。フル画面で初回メッセージを付けて作成した場合のみその先頭文字)を仮置き
  - **初回 AI 応答の完了時に、そのやり取りの内容から AI がタイトルを自動生成して上書き**(無効化スイッチあり / 生成に失敗した場合は暫定タイトルのまま)
  - 受講生による手動編集も可能(1〜100 文字)

### メッセージ送受信(会話オーナーのみ、本文 1〜2000 文字)

- 受講生がメッセージを送信すると、AI 応答を同期的に取得して画面に表示・保存する
- AI への入力には直近の会話履歴(一定件数)を引き渡す(エラー状態のメッセージは除外)
- AI 応答に失敗しても受講生の質問は保持し、AI 応答はエラー状態で保存する(受講生は同じ内容を送り直して再質問できる)

### AI へのコンテキスト自動付与(資格・教材)

受講生が今いる文脈を AI への入力(システムプロンプト)に自動で添える。何が渡るかは受講生の状態で決まる:

| 受講生の状態 | AI に渡るコンテキスト |
|---|---|
| 教材セクションを閲覧中(その画面のウィジェットから起動) | 閲覧中セクションの所在(Part / Chapter / Section の見出し)＋ そのセクションの所属資格名 |
| 教材以外の画面 + デフォルト資格(受講生が既定として選んでいる受講資格)あり | デフォルト資格名のみ(教材の文脈は付かない) |
| 教材以外の画面 + デフォルト資格なし | なし(一般的な学習相談として応答) |

- 資格名は「学習中」または「修了」の資格のみ対象(退会・不合格は付与しない)
- 教材は **見出しのみ**(Part / Chapter / Section の番号 + タイトル)を渡し、セクション本文は渡さない
- 教材セクションのウィジェットから始めた場合、その Section に紐づく既存会話があれば再開し、なければ新規作成する(新規乱立を防ぐ)。フル画面の「新しい相談」からは常に新規作成する
- 教材セクションに紐づく会話は、その資格への受講登録が「学習中 / 修了」であることを整合性チェック

### フローティングウィジェット(学習中受講生のみ表示)

- 学習中受講生の全画面右下に常駐(コーチ / 管理者 / 学習中以外の受講生 / AI 相談画面では非表示)
- セミモーダルで開閉、フル画面遷移ボタンあり
- 開閉状態と表示中の会話 ID をページ遷移後も同タブ内で引き継ぐ

### アクセス制御(機能群共通)

- 学習中受講生のみ AI 相談ルートにアクセス可能
- コーチ / 管理者 / 学習中以外の受講生(招待中 / 修了 / 退会)はアクセス拒否
- 会話のオーナーのみが自分の会話に対する全操作を実行可能(他受講生による直アクセスは拒否)

### 非機能要件

- 機能 OFF スイッチによる全ルート無効化(サイドバー / ウィジェットも非表示)
- 日次送信上限: 受講生 1 人あたり 50 通 / 日(超過時は送信拒否)
- AI API キー未設定時の案内表示
- AI 応答の運用観測メタデータ記録(モデル名 / トークン数 / 応答時間)。受講生には表示せず、運用・障害解析の内部記録に留める

## スコープ外

- ストリーミング応答(SSE)— 同期版のみ採用
- 管理者 / コーチによる AI 相談機能の利用 — 受講生専用
- 他受講生の会話履歴閲覧(管理者 / コーチ含む)— プライバシー / 監査外
- Section 以外(問題演習 / 模試結果 / 質問掲示板等)への会話コンテキスト紐付け
- 完全な RAG(埋め込みベース教材検索)— 教材本文埋め込みは外部 API 無料枠を圧迫するため見送り
- 教材ファイルアップロード API の利用
- システムプロンプトの管理画面 / DB 管理 — 環境設定で完結
- 会話の自動削除 / 一括クリーンアップ — 削除は受講生本人の手動操作のみ
- 受講生間での会話共有 / 会話エクスポート
- 音声入力 / 画像添付 / マルチモーダル
- AI 応答へのフィードバック収集(👍 / 👎)
- AI 失敗時の Rate Limit クォータ補正(失敗分も日次カウント)
- 自前のトークン数切り詰め(履歴件数のみで制御)
- 外部 LLM の複数同梱 — Repository パターンで切替可能性は残すが本チケットでは Gemini のみ実装

## 受け入れ条件

- [ ] 【受講生】最新会話への自動遷移
  1. `/ai-chat` にアクセスした際に、最新の会話の詳細画面に遷移する(会話 0 件の場合は新規相談を促す空状態画面が表示される)
- [ ] 【受講生】会話の作成・再開
  1. フル画面の「新しい相談」から会話を作成した際に、作成された会話の詳細画面へリダイレクトし、フラッシュ「新しい相談を開始しました。」が表示される(フル画面からは常に新規作成。既存会話の再開は教材セクションのウィジェット経由のみで、その場合は画面遷移・フラッシュを介さずウィジェット内で継続する)
  2. 教材セクションのウィジェットから相談を始めた場合は、その Section の既存会話があれば再開され、なければ新規作成される(画面遷移せずウィジェット内で続く)
- [ ] 【受講生】メッセージ送信と AI 応答
  1. メッセージを送信した際に、受講生メッセージが即時表示され、続いて数秒〜十数秒後に AI 応答が会話画面に表示・保存される
  2. 最初の AI 応答完了時に会話タイトルが自動生成されて反映される(タイトル自動生成 OFF の場合は暫定タイトルが保持される)
- [ ] 【受講生】AI 応答失敗時の再送信
  1. AI 応答に失敗した場合、受講生メッセージは保存され、AI 応答はエラー状態で表示される(受講生は同じ内容を送り直して再質問できる。案内「AI が応答できませんでした。しばらく時間をおいて再試行してください。」)
- [ ] 【受講生】会話の削除
  1. 会話を削除した際に、配下のメッセージも連動削除され、削除後の会話 URL を開いても会話は表示されない
- [ ] 【受講生】AI 相談が利用できない条件
  1. 機能スイッチ OFF / API キー未設定 / 受講生の日次送信上限超過では、サイドバー・フローティングウィジェットが非表示(機能 OFF 時)か、利用できない旨の日本語案内が表示され、AI 相談画面の表示・メッセージ送信が行われない(API キー未設定「送信に失敗しました。」/ 日次送信上限超過「本日の利用上限に達しました。明日 0:00 以降に再度ご利用ください。」)
- [ ] 【受講生・コーチ・管理者】フローティングウィジェットの表示
  1. フローティングウィジェットが学習中受講生の全画面に表示される(コーチ / 管理者 / 学習中以外の受講生 / AI 相談画面では非表示)
  2. 開閉状態と表示中の会話 ID がページ遷移後も同タブ内で引き継がれる
- [ ] 【受講生】会話作成 / タイトル更新 / メッセージ送信のバリデーション
  1. 違反時に日本語エラーが表示され保存・送信されない(会話作成 = 教材セクション〈任意・既存セクションと整合〉・初回メッセージ〈任意・1〜2000 文字〉・起動経路〈ウィジェット / フル画面のいずれか〉 / タイトル更新 = タイトル〈必須・1〜100 文字〉 / メッセージ送信 = メッセージ本文〈必須・1〜2000 文字〉)
- [ ] 【受講生・コーチ・管理者】会話のアクセス制御
  1. 会話のオーナーのみが自分の会話を閲覧・編集・削除・メッセージ送受信でき、他の受講生は他人の会話にアクセスできない
  2. コーチ / 管理者 / 学習中以外の受講生(招待中 / 修了 / 退会)は AI 相談ルートにアクセスできない
- [ ] 本チケットの機能に対するテスト (Unit / Feature 等) が実装されている

## 実装方針(参考)

### インターフェース(必須)

**エンドポイント**:

| HTTP | パス | 認可 | 振る舞い |
|---|---|---|---|
| GET | `/ai-chat` | 受講生(学習中)のみ | 最新会話に 302 リダイレクト、0 件なら空状態 View |
| POST | `/ai-chat/conversations` | 受講生(学習中)のみ | 新規会話作成。`Accept: application/json` なら 201(新規) / 200(再開) + JSON、それ以外は 302 リダイレクト + フラッシュ「新しい相談を開始しました。」/「会話を再開しました。」 |
| GET | `/ai-chat/conversations/{conversation}` | 当事者(会話オーナー)のみ | 会話詳細(`Accept: application/json` なら JSON 返却) |
| PATCH | `/ai-chat/conversations/{conversation}` | 当事者(会話オーナー)のみ | タイトル更新、リダイレクト + フラッシュ「タイトルを更新しました。」 |
| DELETE | `/ai-chat/conversations/{conversation}` | 当事者(会話オーナー)のみ | 物理削除 + 連動メッセージ削除、リダイレクト `/ai-chat` + フラッシュ「会話を削除しました。」 |
| POST | `/ai-chat/conversations/{conversation}/messages` | 当事者(会話オーナー)のみ | 同期メッセージ送信、成功 200 / 失敗 502 |

> 認可拒否は基本 403(リソース不在は 404)。コーチ / 管理者 / 学習中以外の受講生はミドルウェアで全ルート拒否、他受講生の会話への直アクセスは Policy で 403。管理者 / コーチ用のバイパスは設けない(他者の会話は監査外)。

**ミドルウェア**: `auth + role:student + active-learning` 全ルート適用 / `throttle:ai-chat`(POST /messages 系、既定 50 通/日)

**機能 OFF 時**: ルート群を未登録にして 404 化(`config('ai-chat.enabled') === false` の場合)。サイドバー項目 / フローティングウィジェットも非表示。

### データモデル

**エンティティ**:

| エンティティ (Model) | 主要属性 | 関係性 | 制約 |
|---|---|---|---|
| AI 相談会話 (`AiChatConversation`) | オーナー / 教材コンテキスト(資格 / Section、任意) / タイトル / 最終メッセージ日時 | User に所属 / Section に関連(任意) / Enrollment に関連(任意) | オーナー FK `cascadeOnDelete` / Section・Enrollment FK `nullOnDelete` / SoftDelete 不採用(物理削除) |
| AI 相談メッセージ (`AiChatMessage`) | 発言者 / 本文 / 状態 / AI 観測メタ(モデル名 / トークン数 / 応答時間 / エラー詳細、受講生には非表示) | 会話に所属 | 会話 FK `cascadeOnDelete`(親会話削除時に連動物理削除) / SoftDelete 不採用 |

**Enum**:

- 発言者 (`AiChatMessageRole`): 受講生 (`User`) / AI (`Assistant`)
- メッセージ状態 (`AiChatMessageStatus`): 処理待ち (`Pending`) / 完了 (`Completed`) / エラー (`Error`)

**インデックス用途**:

- 受講生 × 最終メッセージ日時(一覧降順並び替え)
- 受講生 × Section(既存会話再開判定)
- 会話 × 作成日時(会話内メッセージ取得)

**初期データ**:

- 固定の受講生に複数の会話を投入する（過去の相談の再開・履歴表示を確認）
- 会話の中に、AI 応答がエラー状態のものを含める（エラー表示・同じ内容の再送を確認）
- 教材から始めた会話・教材によらない会話の両方を含める

### コンポーネント

> Action / Service / Repository は Advance 範囲(Basic 受講生は Controller 内完結も可)。

**FormRequest** (`app/Http/Requests/AiChat/`, `app/Http/Requests/AiChatMessage/`)
- `AiChat\StoreRequest` / `AiChat\UpdateRequest` — 会話作成 / タイトル更新
- `AiChatMessage\StoreRequest` — メッセージ送信

**Action** (`app/UseCases/AiChat/`, `app/UseCases/AiChatMessage/`、※ Advance 範囲、Basic 受講生は Controller 内完結も可)
- `AiChat\{Show,Store,Update,Destroy,GenerateTitle}Action` — 会話の各業務操作 + タイトル自動生成

**Service** (`app/Services/`、新規、※ Advance 範囲)
- `AiChatPromptBuilderService` — 資格コンテキスト(会話の資格 ?? デフォルト資格、学習中 / 修了のみ)と教材コンテキスト(Section の Part / Chapter / Section 見出し、本文は除く)を解決してシステムプロンプトに埋め込み

**Repository** (`app/Repositories/`、新規、外部 API 切り離し、※ Advance 範囲)
- `Contracts\LlmRepositoryInterface` + `GeminiLlmRepository` — LLM 抽象 + Gemini 実装(API キー設定 / リクエスト送信 / レスポンスパース / 通信失敗時の例外送出)

**Policy** (`app/Policies/`)
- `AiChatConversationPolicy` — 当事者(オーナー)判定 + 学習中受講生のみ許可。管理者 / コーチ用バイパスは設けず一律拒否

**Model + Enum** (`app/Models/`, `app/Enums/`)
- `AiChatConversation` / `AiChatMessage` / `AiChatMessageRole` / `AiChatMessageStatus`

**View**(既存、ロック対象)

### 異常系

**入力検証**(FormRequest クラス名 + ルール記法):

- 会話作成 (`AiChat\StoreRequest`):
  - `section_id`: `nullable` / `ulid` / `exists:sections,id`
  - `message`: `nullable` / `string` / `min:1` / `max:2000`
  - `source`: `nullable` / `string` / `in:widget,full-screen`
- タイトル更新 (`AiChat\UpdateRequest`):
  - `title`: `required` / `string` / `min:1` / `max:100`
- メッセージ送信 (`AiChatMessage\StoreRequest`):
  - `content`: `required` / `string` / `min:1` / `max:2000`

> 推奨エラーメッセージの日本語文言例は **想定 Q&A セクションに集約**(本セクションは機械的ルール記法のみ)。

**業務例外**(状態ベースガード + HTTP ステータス):

- 教材コンテキストでの会話作成時、所属資格への受講登録が「学習中」または「修了」でない (`AiChatConversationCreationDeniedException`) → 403
- 他人の会話 URL への直アクセス(`AiChatConversationPolicy::view` 拒否) → 403

**外部 API フォールバック**:

- AI 応答失敗時: 受講生メッセージは保存され、AI 応答は `AiChatMessageStatus::Error` で保存し、`AiChatLlmFailedException`(502)を返す(受講生は同じ内容を送り直して再質問できる)
- AI API キー未設定時: `AiChatNotConfiguredException`(500)を投げ、「AI 相談機能は現在ご利用いただけません」エラー画面表示
- 日次送信上限超過時: `AiChatRateLimitExceededException` で上限超過の旨を表示、送信拒否(429)

### 設計判断

- **LLM Repository パターン採用** (`Contracts\LlmRepositoryInterface` + `GeminiLlmRepository`): 外部 API(Gemini)呼出を Repository クラスに抽象化し、Service / Action から API クライアントを直接利用しない。将来 OpenAI 等への切替や Mock 化を容易にする
- **プロンプト組立 Service 分離** (`AiChatPromptBuilderService`): Section / Enrollment 解決 + 教材コンテキスト埋め込みを Action から分離。テスト時の差し替えが容易で、複雑なテンプレ組立を 1 箇所に集約できる
- **同期応答方式**: ストリーミング SSE は非採用。応答待機時間中はローディング UI で UX を保つ
- **日次レート制限**(50 通/日/受講生): 外部 API の無料枠保護 + 過度な依存の抑制
- **会話タイトル AI 自動生成**(`AiChat\GenerateTitleAction`、OFF スイッチ対応): 受講生のタイトル付け負担を削減。初期は暫定タイトル(「新規相談」、または初回メッセージ付きで作成した場合はその先頭文字)とし、最初の AI 応答完了時に上書き。失敗時は暫定タイトル保持で chat 本流を阻害しない
- **メッセージ状態 3 値** (`AiChatMessageStatus::{Pending,Completed,Error}`): AI 応答失敗時にメッセージを `Error` で残すことで、受講生の質問を履歴に保持し、エラー表示のまま同じ内容を送り直して再質問できるようにする
- **資格コンテキストは backend 解決 / 教材コンテキストの供給だけ View Composer** (`AiChatPromptBuilderService` + `SectionPageMetaComposer`, 既存): 資格名は受講生に紐づくユーザー属性でページに依存しないため、プロンプト組立時に backend が「会話の資格 ?? デフォルト資格」で直接解決する(View Composer 非経由、FE への往復不要)。一方、閲覧中セクションはページ依存の文脈で、全画面共通レイアウトに常駐するウィジェットへ「今どのセクションか」を伝える必要があるため、教材セクション画面の View Composer が pageMeta として供給する。供給経路を変えるのはデータのスコープ(ユーザー属性 vs ページ依存)が異なるため
- **テスト観点**: AI 応答失敗 → `Error` 保存 + user メッセージ保持(フォールバック)、教材コンテキスト時の受講登録ガード、Policy の当事者判定が本チケット固有の観点。`LlmRepositoryInterface` は Mock 化して実 API に通さない

## 補足

### 想定 Q&A

| 質問 | 回答 |
|---|---|
| 利用できるロールは? | 受講生(学習中)のみ。コーチ / 管理者 / 学習中以外の受講生は AI 相談ルート全体で 403 |
| AI 応答はストリーミング(逐次表示)? | しない。同期版のみ採用(数秒〜十数秒の待機後にまとめて表示) |
| メッセージの文字数制限は? | 1〜2000 文字。範囲外は 422 で拒否 |
| 1 日に送信できる上限は? | 設定値で既定 50 通 / 日 / 受講生。環境変数で調整可。超過時は 429 |
| AI 失敗時に受講生入力は消える? | 消えない。受講生メッセージは保存され、AI 応答はエラー状態で保存される。受講生は同じ内容を送り直して再質問できる |
| 教材閲覧中にウィジェットを開いたら? | 同 Section の既存会話があれば再開、なければ新規作成。AI への入力に Section の所在(Part / Chapter / Section の見出し)＋ そのセクションの所属資格名が自動付与される |
| 教材閲覧中以外の画面でウィジェットを開いたら? | 教材の文脈は付かない。デフォルト資格を設定していればその資格名のみ AI に渡り、未設定なら資格情報なし。同一タブで直前に開いていた相談があればそれを引き継ぎ、なければ新規作成 |
| AI はどの資格を前提に回答する? | 教材閲覧中はそのセクションの資格、それ以外はデフォルト資格(未設定なら無し)。いずれも「学習中 / 修了」の資格のみ |
| AI への会話履歴は何件渡される? | 設定値で既定 20 件(エラー状態は除外)。環境変数で調整可 |
| 教材本文(Section の長文本文)は AI に渡される? | 渡さない。Part / Chapter / Section の見出し(番号 + タイトル)のみ |
| 認可拒否時の HTTP ステータスは 403 / 404? | 403。他受講生の会話 URL に直アクセスしても 403 |
| 削除は SoftDelete / 物理削除? | 物理削除。配下メッセージも連動物理削除 |
| 機能 OFF スイッチを切ったら? | AI 相談ルート全体が 404、サイドバー項目とフローティングウィジェットも非表示 |
| API キーが未設定の環境では? | リクエスト時に「AI 相談機能は現在ご利用いただけません」が表示される |
| 会話タイトルは誰が決める? | 初期は「新規相談」(フル画面で初回メッセージを付けて作成した場合のみその先頭 30 文字)を暫定とし、最初の AI 応答完了時に AI が自動生成して上書き。タイトル生成 OFF スイッチで無効化可能 |
| タイトル生成が失敗したら? | 暫定タイトルが保持される。チャット本流の応答は阻害されない |
| 会話タイトルの編集の文字数は? | 1〜100 文字。範囲外は 422 |
| 受講生 / コーチ / 管理者は他者の会話を見れる? | 見れない。Policy 拒否で 403。プライバシー / 監査外 |
| AI 観測メタデータ(モデル名 / トークン数 / 応答時間)は受講生に見える? | 見えない。運用・障害解析用の内部記録で、受講生向け画面には表示しない |
| フラッシュ文言の推奨は? | 新規会話「新しい相談を開始しました。」/ 既存再開「会話を再開しました。」/ タイトル更新「タイトルを更新しました。」/ 会話削除「会話を削除しました。」(適切な日本語であれば文言の細部は採点対象外) |
| ページネーションは? | 会話一覧画面は提供しない(最新会話に redirect、会話切替は詳細画面のサイドリスト) |
| 受講生が複数タブで同会話を開いた場合は? | 各タブのセッションストレージは独立。サーバ側は最終メッセージ日時降順で表示するため、リロードで反映される(リアルタイム同期は持たない) |
| Gemini API キーの取得方法は? | Google AI Studio から個人で取得し、`.env` に設定。詳細は `.env.example` のコメントに記載 |
| どの Gemini モデルを使う? | 既定は `gemini-2.5-flash-lite`(GA・低コストで無料枠が手厚い)。`.env` の `GEMINI_MODEL` で変更可。無料枠の上限はモデル・時期で変動するため利用前に AI Studio で確認 |
