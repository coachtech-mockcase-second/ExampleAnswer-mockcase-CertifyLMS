# Certify LMS 100% 要件シート

## 位置づけ

**コーチ向けの 100% 版要件シート**。受講生に渡るのは抽象化した 30% 版のみで、コーチは本シートを参照して受講生からのヒアリングに応答する(30% 版から 100% 版相当の情報を引き出す手助け)。1 チケット = 1 .md ファイルのため、チケット ID・キーワードのファイル内検索で即答できる。本シートを正として **評価シート** と **30% 版要件シート** が派生する(本シートが Single Source of Truth)。

各チケット .md の主な参照箇所:

- **実装方針(参考)** + **補足(想定 Q&A)**(チケット下半分) — ヒアリング応答の主参照箇所
- **受け入れ条件** — 評価シートの採点行と 1:1 対応する原文。採点作業は `../評価シート.md` 側で完結する(採点者が日常的に使うのは評価シート)

| 役割 | 場所 |
|---|---|
| 各チケット定義(本ディレクトリ) | `Story/` `Bug/` `Task/` 配下 |
| 評価シート(本シートから派生) | `../評価シート.md` |
| 受講生配布形 30% 版(本シートから派生) | `../要件シート_詳細度30%/` |

## ディレクトリ構造

```
要件シート_詳細度100%/
├── README.md              # 本ファイル(ナビ + チケット一覧)
├── Story/                 # 14 件(Basic 9 / Advance 5)
├── Bug/                   # 19 件(Basic 16 / Advance 3)
└── Task/                  # 8 件(Basic 2 / Advance 6)
```

**合計 41 件 / 想定総工数 222h**。Basic / Advance はファイル名(`S-B-XX` / `S-A-XX` 等)で識別する(サブディレクトリで階層化しない)。

## ファイル命名規則

`{チケットID}_{Feature略称-連番}.md`(例: `S-B-01_qa-board-01.md` / `B-B-09_content-management-04.md`)。`B` = Basic / `A` = Advance。
