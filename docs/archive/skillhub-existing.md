# 既存SkillHub（LearnHub）要件定義の分析

## 概要
- 現在の名称: LearnHub（ラーンハブ）
- テーマ: オンライン学習プラットフォーム

## 良い点（活かすべき）
1. テーマ（オンライン学習PF）が実務的で理解しやすい
2. 3ロール設計で認可の学習に適している
3. バグ→リファクタ→新機能の段階的な進行
4. カテゴリ管理を「お手本」として提供する設計
5. ブランチ命名規則とPRテンプレートの定義
6. チケットの詳細な実装手順（コーチ向け100%として使える）

## 問題点・変更が必要な点
1. **ボリューム過多**: 93.5h = BookShelf比92%だが、コードリーディング時間を含むと超過する。目標は80%
2. **外部API**: Stripe/SendGridは実務的だが、セットアップが煩雑（APIキー取得等）
3. **Gemini APIチャットボット**: 未含有（ユーザー要件として必要）
4. **Sanctum認証API**: 応用に含まれているが、もっと明確に位置付けが必要
5. **パフォーマンスチューニング**: 応用REFにあるが、体系的に整理が必要
6. **ヒアリング設計**: 受講生向け50%/コーチ向け100%の差分が未設計
7. **チケット数が多すぎ**: 35チケットは管理が煩雑、整理して20-25程度に
