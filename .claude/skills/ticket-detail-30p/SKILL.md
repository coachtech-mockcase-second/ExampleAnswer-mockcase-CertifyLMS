# Certify LMS 詳細度 30% チケット生成 Skill

100% 版チケットを入力に、受講生配布用の 30% 版（仕様を意図的に抽象化）へ変換する。

## 入力

`$ARGUMENTS`: チケット ID（1 件 or カンマ区切り複数件）。なければユーザーに確認。

## 必須読み込み

1. **`.claude/rules/ticket-spec-30p.md`** — 30% 変換規約（paths frontmatter で auto-load、未ロードなら明示 Read）。**書き方・変換ルールはすべてここ**。
2. **入力** = 対応する 100% 版 `関連ドキュメント/要件シート_詳細度100%/{Story|Bug|Task}/{ID}_*.md`。

> `ticket-spec.md`（100% の書き方規約）は読まない。模範解答 PJ / spec も読まない（入力は 100% チケットのみで自己完結）。

## プロセス（各 ID）

1. 100% チケットを Read。
2. 30p 規約の「コア原則・抽象化操作・出力セクション」を適用して変換。
3. `関連ドキュメント/要件シート_詳細度30%/{種別}/{100%と同名}.md` に書き出し。
4. 30p 規約の「禁止」4 点を自己点検（答え漏れ / メタ漏れ / 非一貫 / 回答不能）。

複数 ID 指定時は必須読み込み 1 を 1 回だけ行い、各 ID を順番に処理。

## 完了報告

作成パス + 各チケットで「消した精密仕様」「残した必須サブ節（Story のみ）」を 1 行ずつ。
